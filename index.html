    Authentication

    除Auth/appAuth（请求app_token）接口外，其他所有非登录操作调用接口时均必须在Authorization Header中附上app_token方可取得请求权限，否则将返回404 Not Found。 如果由于更换client_id和secret导致原app_token失效，将返回402 Token Expired。

    若登录成功（包括注册成功、重置密码成功、登录成功），Response Header中的Authorization则被替换为user_token，所有登录操作调用接口时均必须在Authorization Header中附上user_token。 user_token有两种过期时间，分别为access_expire和refresh_expire，均为时间戳，单位为秒，access_expire为7天，refresh_expire为180天。

    如果access_expire已过期而refresh_expire未过期，则返回402 User Token Expired，此时客户端应请求续期接口（Auth/tokenRefresh）获取新的user_token；

    如果refresh_expire已过期（或因为其他原因导致user_token失效），则返回401 Unauthorized，此时客户端应在下一次请求的Request Header的Authorization中提交有效的app_token，此时可执行非登录操作；若仍返回401 Unauthorized，则需要重新登录。

    如果在Request Header的Authorization中提交了有效的app_token（即客户端处于非登录状态），若要执行任何必须登录的操作，相应的接口将返回401。
