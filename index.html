    About Status Code

    所有接口返回的status暂时保留，其含义及对应的status code如下：

    status code 200

    status: 301 status code: 200 // 账号已注册

    status code 400

    status: -400 status code: 400 // 用户请求错误（须提示用户）

    status: -408 status code: 400 // 超过字数限制

    status: -409 status code: 400 // 含有敏感词

    status: -411 status code: 400 // 当天发短信次数用完（一个手机号一天最多发送3次）

    status: -412 status code: 400 // 短信已发送（一分钟内不能重复发送）

    status: -413 status code: 400 // 短信验证码错误

    status: -414 status code: 400 // 用户名含有禁用词

    status: -415 status code: 400 // 账号未注册

    status: -416 status code: 400 // 密码错误

    status: -417 status code: 400 // 邀请码错误或已失效

    status: -403 status code: 403 // 账号被禁言

    status: -406 status code: 403 // 性别审核中

    status: -407 status code: 403 // 性别为男

    status code 401

    status: -401 status code: 401 // 未授权（未登录）

    status code 403

    status: -402 status code: 403 // 未设置性取向

    status: -405 status code: 403 // 性别未审核

    status code 404

    status: -404 status code: 404 // 客户端请求错误（不须提示用户）

    Authentication

    除Auth/appAuth（请求app token）接口外，其他所有非登录操作调用接口时均必须在Authorization Header中附上app token方可取得请求权限，否则将返回404 Not Found。 如果由于更换client_id和secret导致原app token失效，将返回402 Token Expired。

    若登录成功（包括注册成功、重置密码成功、登录成功），Response Header中的Authorization则被替换为user token，所有登录操作调用接口时均必须在Authorization Header中附上user token。 user token有两种过期时间，分别为access expire和refresh expire，均为时间戳，单位为秒，access expire为7天，refresh expire为180天。

    如果access expire已过期而refresh expire未过期，则返回402 User Token Expired，此时客户端应请求续期接口（Auth/tokenRefresh）获取新的user token；

    如果refresh expire已过期（或因为其他原因导致user token失效），则返回401 Unauthorized，此时客户端应在下一次请求的Request Header的Authorization中提交有效的app token，此时可执行非登录操作；若仍返回401 Unauthorized，则需要重新登录。

    如果在Request Header的Authorization中提交了有效的app token（即客户端处于非登录状态），若要执行任何必须登录的操作，相应的接口将返回401。
